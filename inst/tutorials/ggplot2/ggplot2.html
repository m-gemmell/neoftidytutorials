ggplot2

This tutorial covers ggplot2, the visualisation package of Tidyverse. It utilises the grammar of graphics (the gg in ggplot2) to create plots/figures in a layered/compartmentalised fashion.

Sections

This tutorial will contain the following main sections:

A introduction to ggplot2 covering:

  • The creation of a basic plot
  • The format of input data
  • Mapping variables/columns to Aesthetics
  • Building up a plot with components using +

Layers

The different types of plots/geoms including:

  • Bar charts with geom_bar()
  • Histograms with geom_histogram()
  • Line graphs with geom_line()
  • Scatter plots with geom_point()
  • Box and whisker plots with geom_boxplot()
  • Other geoms

Customisations

There are many components that can be used to customise your plots including:

  • Modifying labels with labs()
  • Changing the legend and fonts with theme()
  • Modifying the scale of the x and/or y axis
  • Changing the colours used for aesthetics
  • Changing the guides (i.e. the legend and axis values)
  • Adding annotations on top of plots
  • Modify the x and y axis aspect ratio
  • Facet/split the plot into multiple panels based on variables

Anatomy

There are multiple parts of a ggplot2 figure including:

  • The ggplot2 object that contains the data and aesthetics
  • The aesthetics: The visual properties of the plot that the variables/columns are mapped to (x axis, y-axis, colour etc.)
  • The geom layers: How the aesthetics are visualised (bar chart, boxplot, histogram etc.)
  • The guides: The text and other information on the axes and legends (labels, titles, etc.)
  • Facetting: Splitting of data to multiple plots (e.g. side by side box plots displaying different metrics)

Basic plot

A basic plot consists of:

  • Creating a ggplot object with ggplot() which includes:
    • The input data: a long formatted table, typically in tibble or data.frame format
    • Aesthetic mappings: The parts of a plot variables/columns are mapped to
  • Adding components with +
  • Adding at least one layer/geom to determine how the aesthetics are visualised (e.g a bar chart with geom_bar())

Input data

Input tables (tibbles or data.frames) for ggplot2 must be long formatted.

Long data has the following three features:

  • Each variable is a column; each column is a variable
  • Each observation is a row; each row is an observation
  • Each value is a cell; each cell is a single value

Customisation

A great advantage of ggplot2 is the ability to customise plots by adding components and options.

Components

Various non layer components includes:

  • labs(): Modifying labels on the axes, legend, and other plot labels
  • theme(): Modifying components of a theme including the legend position and the font sizes in the plot
  • Scale values: How to modify the scale of the x and y axis (e.g. plot values as log10 values or square root values)
  • Colour scales: How to set the colours for aesthetics
  • Guides: Modify the appearance of guides including the layout of the legend and axes values
    • Setting number of rows for legend values
    • Allowing axis value/guides to dodge each other when they would otherwise overlap
  • Annotations: Adding text and lines onto the plotting area
    • Includes how to create a statistical lm abline with the equation on the plot
  • Coordinate systems: Modify how the x and y aesthetics combine
    • Modify the x and y aspect ratio
  • Facetting: Splitting a plot into multiple panels by subsetting the visualised data by one or two metadata categories

Other geoms +

Carry out the following:

  • Keep rows where class is equal to “poisonous”
  • Set the bin width of geom_hex() to 2
    • official geom_hex() reference page Tidyverse other geoms page
  • Set the plot aspect ratio to 1
  • Set the continuous x scale with the following features:
    • The name/label is “Stem width (cm)”
    • The breaks go from 0 to 60 by steps of 5
  • Set the continuous y scale with the following features:
    • The name/label is “Stem height (cm)”
    • The breaks go from 0 to 20 by steps of 5
    • The limits go from -1 to 22
  • Set the title label to “Count of stem height against stem width of poisonous mushrooms”
  • Set the legend position to the bottom of the plot
mushroom_tbl |>
  ggplot2::ggplot(aes(x=stem_width, y=stem_height)) +
  ggplot2::geom_hex()
mushroom_tbl |>
  dplyr::filter(class=="poisonous") |>
  ggplot2::ggplot(aes(x=stem_width, y=stem_height)) +
  #Set bin width of geom hex
  ggplot2::geom_hex(binwidth=2) +
  #Fixed aspect ratio
  ggplot2::coord_fixed(ratio=1) +
  #Change x and y scales
  ggplot2::scale_x_continuous(name="Stem width (cm)", breaks=seq(from=0,to=60,by=5)) +
  ggplot2::scale_y_continuous(name="Stem height (cm)", 
                              breaks=seq(from=0,to=20,by=5),limits=c(-1,22)) +
  #Custom labels
  ggplot2::labs(title="Count of stem height against stem width of poisonous mushrooms") +
  #Set plot legend to bottom of plot
  ggplot2::theme(legend.position="bottom")
